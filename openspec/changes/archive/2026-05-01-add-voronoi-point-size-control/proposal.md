## Why

Размер маркеров сайтов (точек) на диаграмме сейчас зашит в коде отрисовки, из‑за чего пользователь не может сделать точки крупнее для наглядности или мельче, чтобы не перекрывать ячейки. Настраиваемый масштаб визуальных точек улучшит читаемость и контроль над презентацией диаграммы.

## What Changes

- Добавить пользовательскую настройку масштаба визуального отображения точек на диаграмме Вороного (относительно текущего базового размера **1×**).
- Ограничить значение диапазоном **0,25×** (минимум) — **1,5×** (максимум); значение по умолчанию **1×**.
- Применять масштаб только к отрисовке точек (круги поверх растровой диаграммы), без изменения геометрии расчёта Вороного и координат сайтов.

## Capabilities

### New Capabilities

- `voronoi-point-visual-size`: Управление масштабом визуальных маркеров сайтов на экране диаграммы с ограничениями 0,25×–1,5× и значением по умолчанию 1×.

### Modified Capabilities

- None.

## Impact

- Affected code: `lib/widgets/voronoi_canvas.dart` (радиусы точек), `lib/controllers/voronoi_controller.dart` и `lib/screens/voronoi_screen.dart` (состояние и UI-контроль), при необходимости тесты виджета/контроллера.
- Формат `.vjson` и нативный расчёт Вороного не затрагиваются, если масштаб остаётся чисто презентационным.
- Dependencies: без новых внешних пакетов (Material 3 / существующая архитектура контроллера).
