## Context

Точки сайтов рисуются поверх растрового изображения Вороного в `VoronoiCanvas`: внешний круг радиусом 6 и внутренний — 4 (логические пиксели холста). Это фиксированные константы; пользователь не может их менять. Требуется ввести коэффициент масштаба **относительно этого базового размера (1×)** с клампом **0,25×–1,5×**. Геометрия FFI и `.vjson` не меняются.

## Goals / Non-Goals

**Goals:**

- Предоставить управляемый пользователем масштаб только для отрисовки маркеров точек.
- Жёстко ограничить ввод и внутреннее значение диапазоном [0,25; 1,5], по умолчанию 1,0.
- Обновление визуала без пересчёта диаграммы (достаточно `setState` / перерисовки `CustomPainter`).

**Non-Goals:**

- Сохранение масштаба в `.vjson` или между перезапусками приложения.
- Изменение размера/формы ячеек или влияние на алгоритм Вороного.
- Отдельные масштабы для обводки и заливки (достаточно единого множителя к обоим базовым радиусам).

## Decisions

1. **Где хранить состояние:** как и смещение оттенка, держать масштаб в состоянии экрана (`VoronoiScreen`), передавать в `VoronoiCanvas` параметром. Это визуальная настройка сессии, не часть `VoronoiDocument`.

2. **Формула отрисовки:** `outerRadius = 6 * scale`, `innerRadius = 4 * scale`, где `scale` уже клампнут. Базовый вид при `scale == 1` совпадает с текущим.

3. **UI-контрол:** `Slider` (или `Slider.adaptive`) с `min: 0.25`, `max: 1.5`, `divisions` опционально для дискретных шагов; отображать текущее значение (например «100 %» или «1,0×») рядом с подписью, по аналогии с блоком оттенка.

4. **Кламп:** при любом вводе (программном или из виджета) применять `clamp(0.25, 1.5)` до передачи в painter, чтобы не нарушать контракт даже при будущих изменениях API.

**Альтернатива (отклонена):** хранить в `VoronoiController` — избыточно, пока настройка не участвует в экспорте и не нужна другим экранам.

## Risks / Trade-offs

- **[Риск]** Очень крупные точки при 1,5× перекрывают мелкие ячейки → **смягчение:** пользователь осознанно выбирает масштаб; диапазон задан продуктом.
- **[Риск]** `shouldRepaint` не учитывает новый параметр → **смягчение:** добавить сравнение масштаба в `shouldRepaint` у `_VoronoiPainter`.

## Migration Plan

Не требуется: изменения только в клиентском UI и отрисовке, откат — revert коммита.
